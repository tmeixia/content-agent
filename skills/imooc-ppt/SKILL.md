# 慕课 PPT 内容 Skill（流水线 Step 3）

把讲师稿转成**逐页 PPT 内容**，严格套用慕课网 PPT 模板规范。产出的是"幻灯片内容描述"，可再交给官方 `pptx` skill 生成最终 .pptx。

## 输入
- 讲师稿 `output/imooc/<topic>/<lesson>/script.md`
- **慕课网 PPT 模板规范** `templates/imooc-ppt-template.md`（慕课官方《课件质量标准》提炼，2026-06-12 用户已确认）

## 输出
- 写到 `output/imooc/<topic>/<lesson>/slides.md`
- 逐页：页类型 + 标题 + 要点 + 备注（对应讲师稿哪段口播）+ 视觉提示

## 核心原则：PPT 跟着讲师稿走
- **每页 PPT 必须对应讲师稿的一个分镜/知识点**，不能各讲各的。
- PPT 是"视觉锚点"，不是讲稿的复制：PPT 放**结论、关键代码、图示**，口播负责展开。
- 一页只讲一个点；代码页只放"要讲的那几行"，不贴整段。

## 步骤
1. 读 `templates/imooc-ppt-template.md`，取页型清单与版式规范（封面/目录/知识点页/代码页/小结页/练习页等）。
2. 按讲师稿分镜顺序，逐段映射成 PPT 页：
   - 【本节目标】→ 封面页 + 目标页
   - 每个【屏幕操作】/知识点 → 1 页知识点页或代码页
   - 【容易错的点】→ 对比页（反例 vs 正例）
   - 【本节小结】→ 小结页
   - 【动手练习】→ 练习页
3. 每页输出：
   ```
   ## Slide N | 页型：代码页
   - 标题：为什么 Child 也重渲染
   - 要点：① 父重渲染默认带动子 ② props 没变也渲染
   - 代码（仅高亮关键行）：console.log('Child render')
   - 讲师备注：对应讲师稿"现场点按钮看 Console"那段
   - 视觉提示：箭头标出 console 输出
   ```
4. 套用模板的版式约束（每页字数上限、代码行数上限、配色/字号——以真模板为准）。
5. 术语对照 `knowledge/_glossary.md`。

## 风格约束（沉淀区——人审后把经验写这里）
- PPT 不是讲稿搬运：放结论/关键代码/图，别整段贴文字。
- 每页与讲师稿分镜严格对应，并在"讲师备注"里写清对应哪段口播。
- 代码页只高亮"要讲的那几行"，过长就拆页或截断。

### 【2026-06-12 官方课件质量标准——硬性约束（用户已确认）】
- **16:9 宽屏**；图文排版**至少 5% 留白**。
- **封面四要素**：阶段标题、每章标题、核心词、`Produced By 讲师昵称`。
- **大标题**：微软雅黑、30 号、加粗、慕课红 `#C9394A`、水平居中。
- **正文**：微软雅黑、20 号、深灰 `#474747`；**重点内容标红 `#C00000`**。
- 文字精炼：关键词、短句呈现，**多用图片/图表/流程图**；图形（箭头）样式全篇统一；规避版权问题。
- 不设"每页行数"硬限制（用户确认去除）；一页 1–2 分钟讲完仅作参考（用户确认弱化）；**不要求动画**（用户确认去除）。
- 全章末尾加轻松收尾页（如"撸起袖子，干~"）。
