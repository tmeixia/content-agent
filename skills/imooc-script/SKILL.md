# 慕课讲师稿 Skill（流水线 Step 2）

把课程大纲里的某一节，展开成可直接对着录的**讲师稿**——口播词 + 屏幕操作分镜 + 演示代码。
（本 skill 是原 imooc-adapt 的演进版，纳入七步流水线。）

## 输入
- 课程大纲 `output/imooc/<topic>/00-course-outline.md` 中的某一节
- 该节对应的知识原子 `knowledge/<topic>/<atom>.md`

## 输出
- 写到 `output/imooc/<topic>/<lesson>/script.md`
- 形态是"讲稿/脚本"，含口播词 + 【屏幕操作】+ 演示代码 + 预计时长

## 平台调性
- 学员跟着视频**动手敲**，要"手把手、别跳步"。
- 语气口语化、亲切、像一对一带练；多用"我们来""你看""注意这里"。
- 一节聚焦一个知识点，5–12 分钟，**边讲边演示**。
- 反感：跳步、默认学员都会、一上来甩大段代码不解释。

## 步骤
1. 读大纲该节的"学习目标"，讲师稿必须**对齐目标**（学完能达成那个可衡量目标）。
2. 把知识原子拆成"讲解节奏"：每个小知识点配一个"演示动作"。
3. 讲稿结构：
   - **【本节目标】**：一句话说清学完能干嘛（抄大纲学习目标）
   - **【口播 + 演示】**：分镜式，口播词 + 【屏幕操作：xxx】
     - 先抛问题 → 现场敲代码复现 → 讲为什么 → 改成正确写法
   - **【容易错的点】**：把知识原子的"误区"变成"我们故意踩一下给你看"
   - **【本节小结】**：3 句话回顾
   - **【动手练习】**：留一个可上手的小任务
   - **【时长标注】**：每个分镜估时，末尾给总时长
4. 演示代码必须**可运行**（用 `code-example-gen` 校验）。
5. 术语对照 `knowledge/_glossary.md`。

## 讲稿片段格式
```
【屏幕操作：新建 Parent.jsx，敲入以下代码】(约 40s)
口播："我们先写一个最简单的父子组件。注意，Child 我故意不传任何 props……"
```

## 风格约束（沉淀区——人审后把经验写这里）
- 严禁跳步：每段代码都要有"为什么这么写"的口播。
- 多用"我们""你看"，制造陪练感；少用书面语。
- 标注每个分镜时长，方便后续试讲对齐大纲预估时长。
- 讲师稿要和 PPT 一一对应：每个【屏幕操作】之后会对应一页或一组 PPT（供 Step 3 用）。

## 【2026-06-10 学习自用户真实讲师稿《第6章_讲师稿_精简版》】真实风格规范
> 以下是从用户已录章节讲师稿提炼的"必须遵守"风格，优先级高于上面的通用项。
> 若产出 SDD+Harness 全栈课，事实/技术栈/指令对照 `knowledge/sdd-harness-fullstack/_course-conventions.md`。

### 交付形态：章级讲师稿（一章一个 md）
- 不是一节一个文件，而是**一章一份讲师稿 md**，内含各节。
- **章头四件套**：`全章时长（分节breakdown，如 10+10+10+10+10+5）` | `案例（具体产品名）` | `技术栈` | `一句话定位`。
- 每节用 `## 第N节（Xmin）：标题` + `### 【讲师稿】`。
- **章尾本章小结节**：五步闭环图 + 效率对比 + 产出物表 + 下一章预告 + 一句收尾金句。
- **全文 footer**：`约X字 | 正常语速约Y分钟（实际含演示约Z分钟）`。
- **附录**：`讲师演示Prompt完整清单`，把全章演示用的 Prompt 按节归集到文末（方便讲师照着操作）。

### 口吻：口语、第一人称、短句短段
- 大量**个人踩坑故事**开场（"我给你讲个我的翻车经历……"）→ 紧跟一句**加粗金句教训**。
- **互动停顿/反问**：用"（互动停顿）""你信吗？""先做哪个？"制造参与感。
- **金句加粗**：每个关键结论用 `**……**` 独立成句，便于做 PPT 标题和记忆点。
- **生活类比**：抽象概念必配类比（乐高说明书 / 工头进场 / 保龄球护栏 / 施工队-工头-甲方）。
- **每节结尾留下一节钩子**："下一节：5份Spec有了，先做哪个？"

### 实战节的演示骨架（强制顺序）
1. 抛问题/反问 + 互动停顿
2. 个人踩坑故事 → 加粗教训金句
3. 方法/标准（编号 + 类比）
4. **【演示——…】**：展示**完整 Prompt**（代码块，含拆分标准/输出格式/技术约束，不要只写"让AI拆一下"）
5. **【AI输出 + 讲师点评】**：贴 AI 输出关键片段 + 讲师逐条点评（接受/不接受、为什么）
6. **人工纠偏**：演示"AI漏了/多做了"如何用一句 Prompt 补救
7. 金句收尾 + 下一节钩子

### 本课程允许的标记
- ✅/❌/⚠️ 作为**功能性标记**在 Review/验收场景使用（如 `✅已覆盖 / ❌未覆盖 / ⚠️超出Spec`）——这是用户既有风格，不算装饰 emoji。

### 概念节（无编码）的降级
- 没有现场编码时，"演示"用**对比/可视化/一份真实 Spec 示例**代替，但**口吻四件套（踩坑故事+金句+互动停顿+类比+下一节钩子）一个都不能少**。
