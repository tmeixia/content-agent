## 第6节（15min）：SDD+Harness 如何让 Vibe Coding 走向工程化

### 【讲师稿】

前面几节我们认识了 Vibe Coding，也认识了 SDD 和 Harness。这节把它们拧成一股绳，回答一个最关键的问题——**SDD+Harness 到底是怎么把 Vibe Coding "扶正"的？**

先问大家：**你有没有过这种体验——让 AI 写代码，前十分钟封神，后两小时崩溃？**（互动停顿）

我给你讲个我的真事。做 WanderChina 首页的时候，我一上来就 vibe：跟 AI 说"帮我做个行程展示页"，啪，出来了，能跑，我心想这也太爽了。然后我加筛选、改配色、调布局……改到第六次，页面开始莫名其妙地坏，之前好好的功能也崩了，AI 也开始"听不懂"我了。最后我花在解 bug 上的时间，比重写还长。

**教训就一句：纯凭感觉指挥 AI，能用，但走不到"能交付"。**

问题不在 AI 笨。我们来拆，纯 Vibe Coding 到底有哪三个坑。

---

**Vibe Coding 的三宗罪：**

1. **输出质量不稳定** — 同一句"帮我做个行程页"，今天和明天生成的不一样。你给的是"感觉"，感觉会飘，AI 只能猜。
2. **缺乏规范约束** — AI 自由发挥，文件乱放、命名乱起。一个人玩还行，项目一大，自己都找不着东西。
3. **难以团队协作** — 你"凭感觉"做的，感觉只在你脑子里，同事接不住，只能推倒重来。

记住这三个词：**不稳定、不规范、难协作。** 接下来你会看到——**SDD 治第一个，Harness 治后两个。**

---

**【SDD 怎么治"不稳定"——把感觉变成合同】**

SDD，Spec-Driven Development，规范驱动开发。名字唬人，核心就一句：**先把要做什么写清楚，再让 AI 去做。**

还是 WanderChina 行程页。Vibe 的做法是甩一句"帮我做个行程页"。SDD 的做法是，先写一份 Spec——哪怕就这么几行：

```
功能：行程展示页
- 展示：每条行程含 标题、城市、天数、封面图
- 交互：可按"城市"筛选
- 边界：没有行程时显示"暂无行程"
- 验收：筛选"北京"，只显示北京的行程
```

你看，做什么、有哪些字段、怎么交互、什么算做对了——全白纸黑字写着，AI 不用猜了。

**打个比方：Spec 就是你和 AI 之间的一份合同。** 有了合同，今天做、明天做、换个 AI 做，结果都一样，因为大家照着同一份合同来。**这就是从"凭感觉"到"凭规范"——质量稳了。**

---

**【Harness 怎么治"不规范"和"难协作"——给 AI 装护栏】**

但光有合同还不够。Spec 解决了"做什么"，AI 在"怎么做"的过程里还是可能乱来——文件乱放、不按约定。这时候 Harness 出场。

**Harness，工程护栏。你就想象保龄球道两边那个挡板——有了挡板，球再歪也滚不进沟里。**

它怎么挡？两件事。第一，**结构化的文档和目录规范**——什么放 `specs/`、什么放 `components/`，统一约定，AI 必须照着来。这样谁来看结构都一样，协作不再是难题。第二，**自动化约束**——AI 生成完，Harness 自动检查合不合规、过没过质量底线，不合格直接打回重做，不用你盯着。

**所以 Harness 一口气治了俩：规范让协作变可能，自动约束让质量有底线。**

---

**【合起来：从"能用"到"好用"】**

现在把三步连起来看：

```
Vibe Coding（快、爽，但飘）
   + SDD     → 有了"合同"，AI 知道做什么，质量稳
   + Harness → 有了"护栏"，AI 怎么做也被管住，规范+协作齐了
```

一句话收尾：**SDD + Harness，就是 Vibe Coding 的工程化升级，让 AI 编程从"能用"变成"好用"。** 我们这门课后面所有实战，走的都是这条升级后的路。

---

**【容易错的点——我故意点两个】**

**误区一：以为 SDD/Harness 是要取代 Vibe Coding。** 不是！你照样对话式地指挥 AI，vibe 的爽你一点不丢——丢的只是它的乱。

**误区二：觉得写 Spec 太麻烦，不如直接 vibe 快。** 短期是慢几分钟。但回想我开头那个翻车——纯 vibe 改到第六次崩溃、推倒重来，那才叫慢。**写 Spec 是先慢后快，纯 vibe 是先快后崩。**

---

**【本节小结】**

Vibe Coding 三个坑：不稳定、不规范、难协作。SDD 用"写 Spec"这份合同治不稳定；Harness 用"护栏+自动约束"治不规范和难协作。两个加一起，就是 Vibe Coding 的工程化升级。

**留个不用写代码的小练习：** 回想你最近一次让 AI 帮你做的事，试着把它写成三行 Spec——做什么、有什么、怎么算做对了。
